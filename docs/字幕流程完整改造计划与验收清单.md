# 字幕识别流程完整改造计划、验收标准与 TODO

## 1. 文档目的

本计划将当前分散的开发目标收束为一条可重复验证的主线：

```text
MFA 全量替代 Qwen3-ForcedAligner 的对等测试
→ 固定两套可靠 ASS 的稳定基线
→ 审计 align → split 的失败根因
→ 分别优化 align 和 split
→ 优化翻译疑点与仅疑点音频复核
→ 优化修改后局部重对齐
→ 建立显式聚合质量门
→ 完整回归并确定默认策略
```

“先完全替代”是指建立真正的 MFA 全量对齐实验路径，让同一份 15 秒 ASR transcript 分别由 MFA 和 Qwen3-ForcedAligner 完成全量对齐，再进行对等比较。在两套可靠 ASS 给出充分证据前，不永久修改生产默认对齐器。

## 2. 最终流程目标

```text
prepare
→ 15 秒日语 ASR
→ 确定性文本清理
→ forced alignment
→ split
→ 断句后处理
→ translate 并标注疑点
→ 仅疑点音频复核
→ 被修改条目重对齐
→ 聚合质量门
→ normalize
→ export
```

最终必须回答：

1. MFA 全量对齐是否稳定优于 Qwen3-ForcedAligner；
2. align 阶段有哪些内容、时间或 token 结构问题；
3. split 阶段为何造成 ASS 匹配分下降或短对白丢失；
4. split prompt、规则切分和后处理各自造成什么影响；
5. 翻译疑点是否能稳定路由真正需要音频复核的条目；
6. 音频复核是否提高质量，是否会引入回归；
7. 哪些参数可以晋升为默认值；
8. 质量门能否阻止有明显内容损失的结果被标记为正式完成。

## 5. 阶段一：MFA 全量替代 Qwen3-ForcedAligner 测试

### 5.1 目标

建立两个互斥的完整结果：

```text
固定 transcript → Qwen3-ForcedAligner 全量对齐
固定 transcript → MFA 全量对齐
```

两个结果随后进入完全相同的 split 和质量评估流程。不得使用“Qwen 失败后才调用 MFA”的混合结果来回答全量替代问题。

### 5.2 实施要求

#### 对齐器接口

提供统一选择参数：

```text
--align-backend qwen|mfa
```

至少接入 `align`、`run` 和 `batch`。实验完成前默认值保持 `qwen`。

#### MFA 正式 backend

正式路径应为：

```text
transcript manifest
→ 生成 MFA corpus
→ 音频片段和 .lab
→ 批量执行 MFA align
→ 解析 TextGrid 或 JSON
→ 转换为统一 token/word 时间结构
→ aligned_segments.json
```

不得把现有 `proofread-realign` 的局部 fallback 简单循环几百次作为全量实现。

#### 批量运行

当前逐条启动 MFA 的耗时约为每条 145–155 秒。全量测试必须按整集建立 corpus，并尽量每集只启动一次 MFA 对齐任务。

第一轮以 15 秒原始 ASR segment 为 MFA corpus 单元。连续语音窗口合并模式作为后续独立实验，不与第一轮同时混用。

#### MFA 输入标准化

`.lab` 文本应保留正确日文，只移除字幕控制符、不可用空白和对齐器无法消费的格式字符。不得擅自改写词义，不得用中文翻译、可靠 ASS 或后续人工校对结果替换初始 transcript。

无法满足最低输入条件的条目记录为 `mfa_input_rejected`，不得静默借用标准答案。

#### 输出和统计

统一 aligned manifest 至少记录：

```json
{
  "alignment_backend": "mfa",
  "alignment_unit": "word",
  "tokens": []
}
```

必须统计：

- 输入条目总数；
- 成功、失败、无输出、`<unk>` 和部分覆盖条目数；
- 文本及字符覆盖率；
- token/word 数；
- 非单调、零时长、负时间和越界条目；
- 运行总耗时及每分钟音频耗时；
- 可获取时记录峰值资源和主要错误。

### 5.4 MFA 策略决策

#### 结论 A：MFA 全量替代 Qwen

两套数据均无关键回退，至少一套明确改善，性能可接受，内容守恒完整。此时才将默认 backend 改为 MFA，并保留 Qwen 兼容或 fallback。

#### 结论 B：Qwen 主对齐，MFA 局部 fallback

MFA 能修复部分 Qwen 失败，但全量指标未稳定胜出，或速度、未登录词问题不适合全量运行。正式顺序为：

```text
Qwen 主对齐
→ 可靠性检查失败
→ MFA 局部重对齐
→ 仍失败则保留原时间并标记疑点
```

#### 结论 C：暂不进入生产流程

两套数据均回退、内容覆盖不足、`<unk>` 严重、短对白明显恶化或无法批量稳定运行时，只保留实验命令。
