# 项目文件与代码整理计划

生成时间：2026-07-14

适用目录：`E:\project\qwen3-asr`

## 1. 目标

本计划的目标不是单纯减少目录数量，而是在不损坏现有功能、实验证据和可复现性的前提下，使项目达到以下状态：

1. 仓库根目录只保留源码、测试、文档、入口和必要配置。
2. 源码、开发工具、运行环境、模型缓存、实验工作区、基准产物和备份具有清晰边界。
3. Git 状态只展示应当评审的源码和文档变化，不被虚拟环境、模型或运行产物淹没。
4. 大型 Python 模块按职责逐步拆分，公共逻辑集中维护，CLI 和数据格式保持兼容。
5. 所有移动、归档和删除操作均有清单、备份、验证和回滚路径。
6. 整理后完整测试、编译、CLI 帮助、关键导入和真实 smoke 均通过。

## 2. 强制安全规则

- 未列入当轮修改清单的文件不得修改。
- 每轮修改已有文件前，备份到 `backups\主题-YYYYMMDD-HHMMSS\`。
- 新文件记录到该轮备份目录的 `NEW_FILES.txt`。
- 不直接删除 `workspaces/`、`backups/`、`benchmarks/`、模型、虚拟环境、输入音视频或导入数据。
- 删除候选必须先生成机器可读清单，记录路径、大小、修改时间、用途和保留理由，再由人工明确批准。
- 不用 Git reset、checkout 等方式覆盖当前未提交改动。
- 不把“已加入 `.gitignore`”理解成“可以删除”。忽略、归档和删除是三种不同操作。
- 中文文件必须以明确 UTF-8 编码读取和验证。
- 代码修改后使用 `.venv312\Scripts\python.exe` 运行编译、针对性测试和完整测试。

## 5. 文件分类规则

### A. 必须进入版本控制

- `qwen_asr/**/*.py`
- `optimizer/**/*.py`
- `optimizer/prompts/**/*.md`
- `tests/test_*.py`
- `docs/` 中当前有效的架构、流程、计划和验收文档
- `scripts/` 中可重复执行且不含凭证的脚本
- 根目录入口、依赖和配置文件
- `benchmarks/` 中的基准定义、摘要、环境说明、复现命令和报告生成脚本

### B. 本地保留但默认忽略

- 虚拟环境、模型缓存和下载工具
- 当前仍用于验证的工作区
- 当前 Goal/P6 的关键证据工作区
- 真实输入音视频和不可公开素材
- 凭证、本地环境变量和机器专属配置

### C. 可归档候选

- 已结束 Goal 且已生成最终摘要的旧工作区
- 已被更新报告完全替代的中间实验目录
- 只用于一次性回滚且超过保留期的备份
- benchmark 重复运行中的音频切片、日志和阶段 manifest
- `tmp_manual_tests/` 中已有正式工作区替代的派生副本

归档建议使用按日期和主题命名的只读压缩包，并同时生成 `manifest.json` 和 SHA-256。归档成功、可解压验证且获得人工确认前，不删除原目录。

### D. 删除候选

- `__pycache__/`、`*.pyc`、`.pytest_cache/`、`.ruff_cache/`
- 明确可重建的空临时目录
- 已归档并经人工批准删除的重复副本
- 明确误生成的零字节根目录文件，如 `$env`，但仍需人工批准

### E. 禁止自动处理

- 当前 Git 修改和未跟踪源码
- `workspaces/`、`backups/` 的整目录
- 模型权重、虚拟环境、真实音视频和导入数据
- 不清楚来源或用途的目录，如根目录 `None`
- 当前完整回归引用的任何证据路径

#### P5-0 被否定实验功能退役

`COMPLETED_REJECTED` 表示实验已经完成且候选不得晋升，不代表实现必须永久留在生产包。对已有充分反证、没有生产调用且会持续增加 CLI、Web、测试和维护成本的实验功能，应优先退役，再继续模块化。

初步处理方向：

| 功能 | 当前结论 | 初步处理方向 |
|---|---|---|
| LLM split 四种模式 | 内容守恒、短应答或稳定性不通过 | 从生产 CLI、Web、prompt 和运行路径删除；只保留历史报告 |
| MFA 全量主对齐 | 双数据集 A/B 不通过 | 从生产默认和组合流水线删除；评估将独立实验器移到 `tools/` |
| MFA proofread local fallback | 默认关闭，仅有局部用途 | 与 MFA 全量路径分离；有独立收益证据才保留，否则单独退役 |
| 无收益 align 参数变体 | 9 个变体均未改善目标指标 | 删除生产 CLI 中只服务失败实验的开关；保留当前有效默认和质量检查 |
| 参数矩阵、A/B、诊断报告器 | 只用于实验复现 | 从生产包移到 `tools/` 或归档，不随正式运行入口加载 |
| 自动发布或绕过质量门 | 未达到自动发布条件 | 不增加绕过路径；继续由 quality-gate 阻止正式产物 |

直接删除必须同时满足：

1. 最终状态有明确 `COMPLETED_REJECTED` 证据。
2. 当前生产默认、PipelineRunner、Web 和正式 CLI 没有调用。
3. 与保留功能没有不可分割的共享实现。
4. 历史报告、指标、命令和结论已保留，不靠运行代码解释旧结果。
5. 删除后不会破坏仍需读取的 manifest、checkpoint 或报告 schema。
6. 已形成精确到文件、符号、CLI 参数和测试的删除清单，并获得路径级批准。
